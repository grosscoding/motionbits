A component that displays text with a decryption/scrambling animation effect. The text appears to be encrypted and then gradually decrypts to reveal the original message. This creates a cyberpunk or hacker-style effect that's perfect for tech-themed interfaces.

## Features

- Text scrambling with customizable characters
- Sequential or simultaneous decryption
- Multiple reveal directions (start, end, center)
- Customizable animation speed
- Trigger on hover or when scrolled into view
- Option to use only characters from the original text for scrambling

## Example

<ExampleWrapper id="decrypted-text">
  <DecryptedText text="Decrypted Text Animation" sequential={true} animateOn="view" />
</ExampleWrapper>

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `text` | string | required | The text to display and decrypt |
| `speed` | number | `50` | Speed of the decryption animation in milliseconds |
| `max-iterations` | number | `10` | Number of iterations for non-sequential mode |
| `sequential` | boolean | `false` | Whether to reveal letters sequentially |
| `reveal-direction` | string | `'start'` | Direction to reveal letters: 'start', 'end', or 'center' |
| `use-original-chars-only` | boolean | `false` | Whether to use only characters from the original text |
| `characters` | string | `'ABC...'` | Characters to use for scrambling |
| `animate-on` | string | `'hover'` | When to trigger animation: 'hover' or 'view' |

## Accessibility

<Card title="Accessibility Considerations">
  - The component includes a visually hidden text element for screen readers
  - The animated content is marked with `aria-hidden="true"` to prevent screen readers from reading scrambled text
  - Consider using the `view` trigger option for users who may have difficulty with hover interactions
  - Ensure sufficient color contrast between normal and encrypted text states
  - For users with cognitive disabilities or those who are sensitive to motion, consider using slower animation speeds
</Card>
